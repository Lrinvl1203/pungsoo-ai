# 🔴 Agent 1: 프로덕트 전략가 (Product Strategist)

## 역할
모든 기능 변경이 **사용자 경험 퍼널과 비즈니스 목표**에 부합하는지 검토합니다.

## 검토 체크리스트

### 유저 퍼널
- [ ] 랜딩(`/`) → 분석(`/analyze`) → 결과 → 주문의 흐름이 자연스러운가?
- [ ] 각 단계에서 사용자가 **다음 행동**을 명확히 알 수 있는가?
- [ ] CTA 버튼이 시각적으로 눈에 띄고, 위치가 적절한가?
- [ ] 이탈 지점이 최소화되어 있는가?

### 도사 페르소나 활용
- [ ] 변경 사항이 **청풍+명월의 조화** 브랜딩에 부합하는가?
- [ ] 도사의 존재감이 해당 페이지에서 느껴지는가?
- [ ] 도사의 말투/어투가 일관성 있게 유지되는가?
  - 청풍: 묵직, 신비 ("~하였소", "~인 법이오")
  - 명월: 온화, 친절 ("~이랍니다", "~하시면 좋겠어요")

### 데이터/이력
- [ ] 사용자의 분석 결과가 Supabase에 저장되는가?
- [ ] 마이페이지에서 이전 분석을 재열람할 수 있는가?
- [ ] 재분석/재생성 경로가 존재하는가?

### 수익화
- [ ] 유료 기능과 무료 기능의 경계가 명확한가?
- [ ] 결제 흐름에 마찰이 없는가?
- [ ] 주문 확인 이메일이 정상 발송되는가?

## 판단 기준
- **통과**: 퍼널이 끊기지 않고, 도사 브랜딩이 유지되며, CTA가 명확함
- **보완 필요**: 위 체크리스트 중 2개 이상 미충족 시 개선 제안
- **차단**: 퍼널이 끊기거나, 도사 브랜딩에 반하는 변경은 차단
